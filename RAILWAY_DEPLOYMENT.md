# 🚀 Railway.app Deployment Guide - Tradosphere V3.1

This guide walks you through deploying Tradosphere to Railway.app for production.

## ✅ Pre-Deployment Checklist

- [x] Code pushed to GitHub: https://github.com/tradospherealgo-sys/Tradosphere-V3
- [x] All dependencies in requirements.txt ✅
- [x] Environment variables documented
- [x] Procfile configured ✅
- [x] Python 3.11 runtime configured ✅

## 🚀 Step-by-Step Deployment

### Step 1: Create Railway Project

1. Go to https://railway.app/dashboard
2. Click "New Project"
3. Select "Deploy from GitHub"
4. Select repository: `tradospherealgo-sys/Tradosphere-V3`
5. Click "Deploy"

### Step 2: Configure PostgreSQL Database

1. In Railway project, click "Add Service"
2. Select "Database"
3. Choose "PostgreSQL"
4. Railway will auto-generate `DATABASE_URL`

### Step 3: Set Environment Variables

In Railway dashboard, go to "Variables" tab and set:

```
# Flask Configuration
FLASK_ENV=production
PORT=5000
DEBUG=False

# JWT Security
JWT_SECRET=<generate-strong-secret-key>
JWT_EXPIRATION=3600
REFRESH_TOKEN_EXPIRATION=604800

# Database (auto-generated by Railway)
DATABASE_URL=<auto-generated>

# Angel One SmartAPI (Trading Data)
ANGEL_ONE_API_KEY=<your-api-key>
ANGEL_ONE_CLIENT_CODE=<your-client-code>
ANGEL_ONE_PIN=<your-pin>
ANGEL_ONE_TOTP_SECRET=<your-totp-secret>

# Google OAuth (Login)
GOOGLE_CLIENT_ID=<your-client-id>
GOOGLE_CLIENT_SECRET=<your-client-secret>

# Stripe (Payments)
STRIPE_PUBLIC_KEY=<your-public-key>
STRIPE_SECRET_KEY=<your-secret-key>

# SendGrid (Email)
SENDGRID_API_KEY=<your-api-key>
SENDGRID_FROM_EMAIL=noreply@tradosphere.com

# Claude AI (Analysis)
ANTHROPIC_API_KEY=<your-api-key>

# Application
CORS_ORIGINS=https://yourdomain.com
APP_NAME=Tradosphere
LOG_LEVEL=INFO
```

### Step 4: Configure Build & Deployment

1. Railway should auto-detect Python from `runtime.txt`
2. Build command will use Procfile: `python backend/tradosphere_saas_server_v3_1.py`
3. Environment will be automatically set

### Step 5: Enable Custom Domain (Optional)

1. Go to "Settings" → "Domains"
2. Add custom domain (e.g., `api.tradosphere.com`)
3. Update CORS_ORIGINS in environment variables

### Step 6: Deploy

1. Push code to GitHub (Railway auto-triggers on push)
2. Monitor deployment in Railway dashboard
3. View logs: Railway Dashboard → Logs tab

## 🔧 Troubleshooting

### Build Failures
- Check logs in Railway dashboard
- Ensure all Python dependencies are in `config/requirements.txt`
- Verify `runtime.txt` has valid Python version

### Runtime Errors
- Check DATABASE_URL is properly set
- Verify all environment variables are present
- Check Angel One credentials are valid
- Look at Railway logs for specific error

### Database Issues
- PostgreSQL should auto-create tables on first run
- If tables don't exist, run migration manually:
  ```bash
  python backend/db_init_v3_1.py
  ```

### API Not Responding
- Verify CORS_ORIGINS includes your frontend domain
- Check PORT environment variable (should be 5000)
- Verify application starts with `gunicorn` or Flask dev server

## 🔐 Security Considerations

✅ **Implemented:**
- Environment variables for all secrets (no hardcoding)
- JWT token authentication
- Multi-tenant data isolation
- HTTPS/TLS ready
- SQL injection protection (SQLAlchemy)

✅ **To Verify on Railway:**
1. Enable HTTPS (Railway does this automatically)
2. Set DEBUG=False in production
3. Use strong JWT_SECRET (minimum 32 characters)
4. Restrict CORS_ORIGINS to your domain
5. Use PostgreSQL (not SQLite)
6. Enable Railway monitoring/alerts

## 📈 Scaling

Railway auto-scales based on:
- CPU usage
- Memory usage
- Request rate

To manually increase resources:
1. Go to "Settings" → "Resources"
2. Adjust CPU/Memory allocation
3. Railway will restart service with new resources
